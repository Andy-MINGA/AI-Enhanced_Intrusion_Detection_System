ATTACK TYPE: DoS Hulk
─────────────────────────────────────────────────────────────────
Threat Summary: The DoS Hulk attack is a type of denial-of-service attack that aims to overwhelm a target server by sending a large number of requests, causing it to become unresponsive. This attack exploits the server's resources, leading to service disruption for legitimate users.
Technique: Denial of Service (DoS)
How It Works: The attacker sends a flood of TCP packets with the reset flag set, which can exhaust the target's resources and lead to service unavailability. This is done by manipulating the TCP handshake process to create excessive load on the server.
CVE/CWE: No specific CVE; this attack does not exploit a specific vulnerability but rather targets resource exhaustion.
Severity: Critical - This attack can render services unavailable, impacting business operations and user access.
IOCs: 1. Unusually high traffic volume to the target IP; 2. Multiple TCP reset packets from the source IP; 3. Service unavailability or degradation on the target server.
Action Required: 1. Implement rate limiting on the affected server; 2. Monitor network traffic for unusual patterns; 3. Configure firewall rules to block suspicious IPs; 4. Increase server resources temporarily to handle the load.
Affected Systems: Web servers, application servers, and any services exposed on port 80.

ATTACK TYPE: DoS GoldenEye
─────────────────────────────────────────────────────────────────
Threat Summary: The DoS GoldenEye attack is a type of denial-of-service attack that aims to overwhelm a target system by sending a large number of requests, causing it to become unresponsive. This attack can significantly disrupt services and degrade performance for legitimate users.
Technique: TCP Timestamp Manipulation
How It Works: The attack exploits vulnerabilities in the TCP protocol by sending packets with manipulated timestamps that fall outside the expected range. This can confuse the target system's ability to manage connections, leading to service disruption.
CVE/CWE: No specific CVE - this attack leverages protocol weaknesses rather than a specific software vulnerability.
Severity: Critical - this attack can render services unavailable, impacting business operations and user access.
IOCs: 1. Unusual spikes in TCP traffic from a single source IP; 2. TCP packets with timestamps outside the expected range; 3. Increased connection timeouts or failures on the target system.
Action Required: 1. Implement rate limiting on the affected service; 2. Monitor network traffic for unusual patterns; 3. Update firewall rules to block suspicious IP addresses; 4. Conduct a thorough review of system logs for further indicators.
Affected Systems: Web servers and any services exposed on port 80.

ATTACK TYPE: DoS slowloris
─────────────────────────────────────────────────────────────────
Threat Summary: A Slowloris attack is a type of Denial of Service (DoS) attack that aims to keep many connections to the target web server open and hold them open as long as possible. This prevents legitimate users from accessing the service by exhausting the server's resources.
Technique: Application Layer DoS Attack
How It Works: The attacker sends partial HTTP requests to the server, never completing them, which keeps the connections alive. This gradually consumes the server's resources, leading to denial of service for legitimate users.
CVE/CWE: No specific CVE; this attack exploits the way web servers handle connections rather than a specific vulnerability.
Severity: Critical - This attack can render web services unavailable, impacting business operations and user access.
IOCs: 1. High number of incomplete TCP connections from a single source IP. 2. Unusual patterns of small TCP segments being sent to the web server. 3. Increased latency and unresponsiveness of the web application.
Action Required: 1. Implement rate limiting on incoming connections to the web server. 2. Monitor and analyze traffic patterns for anomalies. 3. Configure web application firewalls to detect and block Slowloris traffic. 4. Increase server resources temporarily to handle potential spikes in traffic.
Affected Systems: Web servers, particularly those running HTTP services on port 80.

ATTACK TYPE: DoS Slowhttptest
─────────────────────────────────────────────────────────────────
Threat Summary: The DoS Slowhttptest attack aims to exhaust server resources by sending partial HTTP requests, causing the server to become unresponsive. This can lead to service downtime and affect legitimate users trying to access the target system.
Technique: Denial of Service (DoS)
How It Works: The attacker sends slow, incomplete HTTP requests to the server, keeping connections open for an extended period. This ties up server resources, making it difficult for legitimate requests to be processed.
CVE/CWE: No specific CVE; this attack exploits the server's resource management rather than a specific vulnerability.
Severity: High - This attack can significantly disrupt service availability and impact business operations.
IOCs: 1. Unusually high number of incomplete HTTP requests; 2. Increased server resource utilization; 3. Slow response times for legitimate users.
Action Required: 1. Implement rate limiting on incoming HTTP requests; 2. Monitor server logs for unusual traffic patterns; 3. Configure firewalls to detect and block suspicious traffic; 4. Increase server capacity to handle potential spikes in traffic.
Affected Systems: Web servers and applications that handle HTTP requests.

ATTACK TYPE: Heartbleed
─────────────────────────────────────────────────────────────────
Threat Summary: Heartbleed is a serious vulnerability in the OpenSSL cryptographic software library that allows attackers to read sensitive data from the memory of affected systems. This can lead to the exposure of private keys, user passwords, and other confidential information.
Technique: Buffer Over-read
How It Works: The attack exploits a flaw in the OpenSSL implementation of the TLS/DTLS heartbeat extension, allowing an attacker to send a malformed heartbeat request and receive more data than intended from the server's memory.
CVE/CWE: CVE-2014-0160
Severity: Critical - This vulnerability can lead to significant data breaches and compromise of sensitive information.
IOCs: Unusual TCP traffic patterns, especially consecutive small TCP segments, and high confidence alerts from intrusion detection systems indicating Heartbleed activity.
Action Required: 1. Immediately patch affected systems with the latest OpenSSL updates. 2. Rotate SSL/TLS certificates and keys to mitigate potential exposure. 3. Monitor network traffic for unusual patterns that may indicate ongoing exploitation attempts. 4. Educate staff on the importance of secure configurations and regular updates.
Affected Systems: Any system using vulnerable versions of OpenSSL, particularly web servers and services that rely on SSL/TLS for secure communications.
